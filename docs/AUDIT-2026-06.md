# Vibemesh-AI — Engineering Audit (2026-06-20)

> Multi-discipline audit by an 11-lead team (108 agents). Every finding was independently
> verified against the real code — **92 findings, 0 refuted**. Health at audit time: clean tree
> on `main`, lint clean, build clean, 134/134 unit tests pass.
>
> Severity (post-verification): **6 high · 31 medium · 48 low · 7 info**.
> Domain maturity: geometry 8, ai-quality 8, security 8, perf 8, product 8, architecture 7,
> frontend 7, uiux 7, state 7, devex-docs 7, testing 6.

## 1. Executive summary

Vibemesh-AI is a notably mature, well-architected single-author codebase: the hardest parts — the
openscad-wasm worker facade, the AI generation/skills/bench machinery, and the geometry contract —
are genuinely production-grade. The **single biggest risk is a cluster of silent-failure and
wrong-context-write bugs on the hot generation path**: pressing Stop during Best-of-N restarts
generation instead of stopping it, a mid-generation project switch writes the reply onto the *wrong*
project, and truncated/aborted streams are adopted as successful replies. The **highest-leverage
opportunity is closing the CI/test gap on the three most intricate untested seams** (the worker
facade, the entire Express server, the store compile lifecycle) plus running the zero-API bench
ratchets in CI — all pure logic, all cheap, all currently capable of merging a regression green. A
recurring secondary theme is **doc/config drift**, and a set of **multi-user/shippable-readiness
gaps** (no focus traps, no SSE heartbeat, no rate limiting, cross-tab clobbering) that are correctly
out of scope for today's single-user local use but block the stated shippable ambition. Nothing here
is a data-loss bug on the single-user happy path.

## 4. Cross-cutting themes

- **Silent failure on the generation channel.** Aborts (Best-of-N), truncation (max_tokens), premature stream close, and mid-generation project switch all masquerade as success. The happy path is well-guarded; the failure paths quietly do the wrong thing.
- **Strong test *quality*, wrong test *placement*.** 134 high-signal assertions and an exemplary gate selftest — yet the three most failure-prone seams had zero coverage despite being pure/mockable, and the bench ratchet never ran in CI.
- **Two god-modules concentrate coupling** — `store.ts` (1545 LOC) and `Viewport.tsx` (1283 LOC).
- **Per-token re-render cascades during streaming** — surfaced independently by Frontend and Performance.
- **Doc/config drift erodes trust in precise-looking numbers** — CONTRIBUTING, CLAUDE.md counts, `.nvmrc` 22 vs `engines >=20`.
- **Single-user-correct, multi-user-incomplete** — a deliberate boundary; the cost is concentrated, not scattered.

## 5. What's working well (protect this)

- The openscad-wasm worker facade is production-grade concurrency control (latest-wins coalescing with a `superseded` sentinel, never-dropped FIFO bg queue, per-job watchdog, respawn-*before*-finish).
- THREE resource disposal is careful where most apps leak (every geometry, PMREM env map, RoomEnvironment disposed; `frameloop="demand"`).
- The bench is a genuine three-valued ratchet (PASS/REGRESSION/INCONCLUSIVE) with transport-vs-generation classification and its own zero-API selftest.
- The skills/retrieval/hardware machinery is rigorous (validators with "exemplar poison" controls, a scored deterministic router, single-source hardware catalog with a numeric drift selftest).
- Security threaded deliberately for local-first (no `dangerouslySetInnerHTML`, WASM sandbox, real CSP in the build, loopback bind, injection-rejecting `.env` writes, honest SECURITY.md).
- Excellent zustand + TypeScript hygiene (atomic primitive-returning selectors, ~zero `any`/`ts-ignore`, well-reasoned compile-lifecycle guards).
- Data-loss-on-export is a first-class contract (HUD note + loud toast for every degraded/failed/oversize piece, per SPEC §4).

## 7. Blind spots the audit itself flagged (uncovered ground)

- **`deploy.yml` is fully decoupled from `ci.yml`** — a green Pages site can ship from a red-CI commit (see P1 #13).
- **`openscad-wasm` is a caret-pinned `^0.0.4` from an unofficial individual maintainer** — the entire geometry engine is one unofficial pre-release tarball; `npm install` (not `npm ci`) in deploy weakens the lock's integrity guarantee.
- **No offline/PWA story despite the "local-first" claim** — no service worker / manifest; a refresh with no network fails to load the shell.
- **Render-blocking Google Fonts CDN** in `index.html` leaks every visitor's IP to Google (contradicts the privacy posture) and breaks offline; no SRI on the stylesheet.
- **Zero observability / error telemetry** — a WASM crash or provider 500 in a deployed demo leaves no trace.
- **Browser/hardware compat of wasm + IndexedDB + WebGL2 is untested** — e2e is chromium-only; no feature-detection for Safari/iOS quirks.
- **Cold-start on a throttled connection is unmeasured** — ~14MB wasm chunk + 1.44MB app chunk, no audited WASM-fetch progress/timeout UX.
- **`public/` has only a favicon** — no `robots.txt`, no OG/social meta, no SPA `404.html` fallback (breaks deep links on Pages).
- **i18n absent and unscoped** — all UI + screen-reader copy is hardcoded English.
